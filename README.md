# ⚡ BOJ Never Die - Chrome Extension

백준(BOJ) 사이트 종료에 대비하여 언제든 문제 페이지에서 코드를 즉시 채점하고, **AI 기반 히든 데이터 생성** 및 **커뮤니티 반례 공유**까지 한 번에 할 수 있는 크롬 익스텐션입니다.

---

## 📦 주요 기능 개요

| 기능 | 설명 | 비고 |
|------|------|------|
| **통합 채점 엔진** | Wandbox API를 기반으로 100% 무료 코드 실행 (C++, Python, Java 등 지원) | |
| **AI 멀티모델 폴백** | Claude, Gemini, OpenAI 지원 및 **사용량 한도 초과 시 하위 모델(Lite 등)로 자동 무중단 폴백(Fallback) 기능** | 사용자 BYOK |
| **강력한 에디터 UX** | 들여쓰기(Tab), 중괄호 자동완성, 클립보드 붙여넣기 등 IDE 수준의 편의성 | |
| **One-Click 채점** | 백준 예제 + AI 히든 케이스 + 커뮤니티 케이스들을 **순차 통합 채점** | |
| **서버리스 커뮤니티 DB** | Edge Function 기반의 안전한 데이터 공유 | |

## 🚀 아키텍처 및 보안 (권장 구조)

본 프로젝트는 오픈소스 공개 및 배포를 위해 **보안 분리 구조(Serverless Architecture)**를 채택했습니다.

### 1. 보안 핵심 (Supabase Edge Functions)
익스텐션(프론트엔드)에서는 Supabase DB로 직접 접근하지 않습니다. 모든 API Key와 인증 헤더를 클라이언트에서 제거하였으며 오직 데이터 접근용 단일 URL(Edge Function)만 사용합니다.

- **Frontend**: 오로지 `EDGE_FUNCTION_URL` 하나로 Unauthenticated GET/POST 요청을 보냅니다. 소스 코드를 깃허브(Public)에 공개해도 데이터베이스 공격 우려가 없습니다.
- **Backend (Deno Edge Function)**: 사용자로부터 들어오는 텍스트 길이나 스팸 등을 **사전 검증(Validation)** 한 뒤, 함수 내부에서만 가지고 있는 `SERVICE_ROLE_KEY`를 사용해 DB의 RLS를 우회하여 데이터를 전달합니다.
- **Database**: 모든 RLS 정책(공개 쓰기/읽기)은 삭제되어 외부와의 직접 연결은 100% 차단됩니다.

### 2. AI 자동 우회 (폭포수 폴백) 전략
AI 모듈이 과부하(HTTP 429) 상태이거나 한도 초과(Quota Error)가 날 경우 익스텐션이 코드를 중단하지 않습니다. 대신 개발된 내부 로직이 자동으로 비용이 더 저렴하거나 안정적인 대체 모델로 투명하게 넘겨줍니다. 
*(예: `gemini-2.5-flash` ➡️ `gemini-2.5-pro` ➡️ `gemma-4-31b`)*

## 👤 일반 사용자 (User) 세팅 가이드

사용자는 확장 프로그램을 스토어에서 다운로드 받은 뒤, 다음 과정을 진행할 수 있습니다.
1. 문제 영역에서 `채점하기(▶)` 버튼을 눌러 내 코드를 무료로 검증합니다.
2. 설정 탭에 진입하여 자신이 소유한 AI API Key (Claude, Gemini, OpenAI)를 하나 기입합니다.
3. 이제부터 히든 케이스 탭에서 AI 히든 케이스 `✨ 생성` 버튼 클릭 시 AI 모델이 문제에 숨겨진 반례나 크기가 큰 경계값 테스트 케이스를 추론해 줍니다. 
    - (AI가 생성한 히든 테스트는 오류가 있을 수 있습니다.)
4. 커뮤니티 케이스의 `불러오기` 버튼을 클릭하면 다른 사용자들이 공유한 히든 테스트 케이스를 불러올 수 있습니다.
5. 불러온 모든 히든 케이스는 채점 탭에서 기본 예제와 함께 테스트 할 수 있습니다. 
6. 추가할 히든 케이스가 있거나 피드백이 있다면 피드백 탭을 활용해 주세요.
